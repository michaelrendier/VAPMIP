# PART III — PYTHON ENHANCEMENT PROPOSALS
# The design decisions behind the language features monad.py uses.
# ═══════════════════════════════════════════════════════════════════
#
# The Zen of Python. 19 aphorisms that define the design aesthetic.
# "There should be one obvious way to do it."
# monad.py follows this: one commit() path, one learn() path.
[PEP]  https://peps.python.org/pep-0020/
#
# Style guide. monad.py follows PEP 8 throughout.
# Underscore prefix: _protected. Double underscore: __private.
# The naming convention is the namespace encoding.
[PEP]  https://peps.python.org/pep-0008/
#
# Python 3 rationale. Why backward compatibility was broken.
# This is the design philosophy document for the language Holcus runs in.
# "Reduce feature duplication by removing old ways."
[PEP]  https://peps.python.org/pep-3000/
#
# Simple generators. yield turns a function into a generator.
# study.py uses generators for corpus iteration.
[PEP]  https://peps.python.org/pep-0255/
#
# Iterators (__iter__, __next__). The iteration protocol.
# Every for loop in monad.py uses this protocol.
[PEP]  https://peps.python.org/pep-0234/
#
# Generator expressions. (x for x in seq if cond)
# Used in monad.py for field statistics: sorted(c._beta[k] for k in range(c.n))
[PEP]  https://peps.python.org/pep-0289/
#
# Coroutines via generators. yield as receive + send.
# The foundation of Python's async model. study.py's threads
# are not async, but understanding this PEP explains why.
[PEP]  https://peps.python.org/pep-0342/
#
# yield from. Delegation to sub-generators.
# Direct predecessor to async/await.
[PEP]  https://peps.python.org/pep-0380/
#
# Decorators for functions and methods. @property, @staticmethod.
# Engine._tier is a @property. MonadInterface uses @property.
# Understanding decorators means understanding what @property
# does to Engine._tier's lookup — it makes the tier computation
# happen at access time, not at definition time.
[PEP]  https://peps.python.org/pep-0318/
#
# The with statement and context managers. __enter__, __exit__.
# monad.py's _RWLock uses with extensively.
# with self._rwlock.writing(): is a context manager invocation.
[PEP]  https://peps.python.org/pep-0343/
#
# Abstract Base Classes (ABCs). abc.ABC, abc.abstractmethod.
# ABCs are used in monad.py to define the MonadInterface contract.
[PEP]  https://peps.python.org/pep-3119/
#
# Function annotations: def f(x: int) -> str
# The type annotations throughout monad.py (Dict, List, etc.)
# are function annotations. They are documentation — not enforced
# at runtime in CPython — but they communicate intent.
[PEP]  https://peps.python.org/pep-3107/
#
# Type hints. typing.Dict, typing.List, typing.Optional.
# The type annotations in monad.py use this system.
# Holcus's own codebase is annotated throughout.
[PEP]  https://peps.python.org/pep-0484/
#
# Variable annotations: x: int = 5
# The class body annotations in Crank.__init__ use this form.
[PEP]  https://peps.python.org/pep-0526/
#
# Structural subtyping (Protocols). typing.Protocol.
# Protocols define interfaces without inheritance — duck typing
# made explicit. Used for the callback signature in study.py.
[PEP]  https://peps.python.org/pep-0544/
#
# Dataclasses. @dataclass generates __init__, __repr__, __eq__.
# The CorpusState in SeedService.kt is a Kotlin data class;
# the Python equivalent would be a dataclass.
[PEP]  https://peps.python.org/pep-0557/
#
# f-strings. f"vocab={c.n}". Literal string interpolation.
# monad.py uses f-strings throughout for logging and display.
[PEP]  https://peps.python.org/pep-0498/
#
# Walrus operator :=. Assignment expression inside conditions.
# Used in monad.py: if (r := engine.generate(prompt)): ...
[PEP]  https://peps.python.org/pep-0572/
#
# Structural pattern matching (match/case). Python 3.10+.
# Used in socket dispatch in monad.py for message routing.
[PEP]  https://peps.python.org/pep-0634/
#
# Pattern matching tutorial. Practical examples of match/case.
[PEP]  https://peps.python.org/pep-0636/
#
# Pathlib. pathlib.Path object-oriented filesystem paths.
# monad.py uses pathlib.Path for .bin file path resolution.
[PEP]  https://peps.python.org/pep-0428/
#
# Virtual environments. venv, site-packages isolation.
# The Chaquopy Python runtime in the APK is an embedded venv.
[PEP]  https://peps.python.org/pep-0405/
#
# Import hooks and the importlib machinery.
# Understanding this explains how Chaquopy makes Python imports
# work inside the Android APK environment.
[PEP]  https://peps.python.org/pep-0302/
#
# super() without arguments. super() in Python 3.
# Used throughout monad.py's class hierarchy.
[PEP]  https://peps.python.org/pep-3135/
#
# Extended unpacking. a, *b, c = seq
# Used in monad.py for tuple destructuring in socket dispatch.
[PEP]  https://peps.python.org/pep-3132/
#
# async/await. Asynchronous I/O via coroutines.
# monad.py uses threads, not async — but understanding this PEP
# explains why. The threading model was chosen deliberately over async.
[PEP]  https://peps.python.org/pep-0492/
#
# Generic aliases: list[int] instead of typing.List[int]. Python 3.9+.
# monad.py still uses typing.List for 3.8 compat; this is why.
[PEP]  https://peps.python.org/pep-0585/
#
# Union types: int | str instead of Union[int, str]. Python 3.10+.
[PEP]  https://peps.python.org/pep-0604/
#
# Rich comparisons. __lt__, __le__, __gt__, __ge__, __eq__.
# The zero-index ordering in monad.py's render path uses rich comparison.
[PEP]  https://peps.python.org/pep-0207/
#
# Integer division. // floor division, / true division.
# monad.py uses both: // for index arithmetic, / for ratios.
[PEP]  https://peps.python.org/pep-0238/
